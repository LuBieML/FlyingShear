MOVELINK(distance, link_dist, link_acc, link_dec, link_axis)

VELOCITY VS MASTER DISTANCE

Optional link_options: S-curves, MARK/MARKB/R_MARK starts, repeat behavior, direction gating, DPOS sourcing. Alias: ML().
Full syntax MOVELINK(distance, link_dist, link_acc, link_dec, link_axis[, link_options[, link_pos[, base_dist]]])
Rules Link distances are positive. Default link feedback is MPOS; option bit 13 uses DPOS. If link_acc + link_dec exceeds link_dist, both are scaled down.
DESCRIPTION
hover a parameter
Each parameter maps to a piece of the motion. distance is what the shear travels; link_dist is the master travel that drives it; link_acc and link_dec shape the ramps.

RETRACT PEAK

How the return speed is calculated

PROFILE FACTOR 4 / 3

Why peak return speed is 4 / 3 × average speed

ACCEL AREA
1/2 × 1/4 = 1/8
PEAK AREA
1/2 = 4/8
DECEL AREA
1/2 × 1/4 = 1/8
TOTAL AREA
6/8 = 3/4
Area under velocity over time is distance. With total return time normalised to 1, that area is also the average velocity. This trapezoid area is 3/4 of the peak speed.
v_avg = 3/4 × v_peak   ==>   v_peak = 4/3 × v_avg
